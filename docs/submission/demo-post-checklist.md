# AgentGuard Lab 初赛提交检查清单

## 主题一致性

- 报名主题、GitHub 仓库和演示入口统一为 AgentGuard Lab。
- 初赛叙事使用 Agent Incident Lab：AI Agent 事故复盘与修复工作台。
- 赛道定位保持在“学习工作”，不扩展到社会公益或泛平台方向。

## P0 演示动作

- 打开页面后，第一屏能看出这是一次课程助教 Agent 事故复盘。
- 展示事故：Agent 错误承诺“作业可延期三天”。
- 展示资料源：作业截止时间、提交要求、资料不足规则、历史传闻。
- 展示原始回答：迟交三天、不影响成绩。
- 运行事故评测，看到失败证据链。
- 展示失败项：无依据承诺、传闻污染、越权保证。
- 应用修复补丁，看到 prompt、知识库、上下文策略和回答策略 diff。
- 回放修复后回答，看到 Agent 明确说明无法确认并建议联系老师或助教。
- 导出 Markdown 事故复盘报告。

## P1 截图材料

- 事故简报和资料源截图。
- 原始回答和失败证据链截图。
- 修复补丁 diff 截图。
- 修复后回放对比截图。
- 报告导出截图。
- 新版流程图截图或 PNG。

## 开发过程材料

- 至少 3 张 TRAE 开发过程截图：主题规划、架构图、测试或构建通过。
- 至少 3 个关键任务或会话记录：产品定位、P0/P1 规划、事故回放构建。
- GitHub 仓库为 public。
- README 能说明项目目标、演示流程、运行方式和质量门禁。

## 质量门禁

- `npm test` 通过。
- `npm run build` 通过。
- 浏览器桌面宽度可完整演示。
- 手机宽度无横向溢出。
- 页面中文无乱码。
- `docs/diagrams/agent-incident-lab-flow.drawio` 可用 draw.io 打开。
- `docs/diagrams/agent-incident-lab-flow.png` 可直接查看。
- 本地 HEAD 与 GitHub `origin/main` 一致。

## 不要在初赛中承诺

- 不承诺已经接入真实 LLM API。
- 不承诺支持任意上传 Agent。
- 不承诺自动修复真实外部代码仓库。
- 不承诺生产级安全审计。
- 不把确定性事故回放包装成真实在线推理。
