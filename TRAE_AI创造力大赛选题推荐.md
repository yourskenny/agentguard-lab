# TRAE AI 创造力大赛选题推荐

调研日期：2026-06-22。目标：根据 yourskenny.top、GitHub 公开仓库、已有 Agent 工程方向，以及 TRAE 比赛评分方式，确定一个更有技术含量、更容易形成差异化的参赛题。

## 最终主推题目

题目名称：**AgentGuard Lab：AI Agent 可靠性评测与修复工作台**

推荐赛道：**学习工作**

不建议强行叠加“社会公益”。这个题更适合按开发者工具、AI 应用质量保障、Agent 工程化工作台来讲，重点放在真实工程价值和可验证闭环。

## 一句话介绍

AgentGuard Lab 是一个面向 AI Agent 开发者的评测与修复工作台。用户输入 Agent 的系统提示词、知识库片段、工具说明或对话记录后，系统自动生成评测用例，运行可靠性测试，定位失败原因，并给出可复测的修复建议。

## 为什么放弃轻量 ProjectLens

上一版 ProjectLens 如果只做“读取 GitHub / 个人网站并生成报名提案”，容易变成包装工具，技术门槛和差异化都不够。它可以作为 AgentGuard Lab 的一个测试样例，但不适合作为冲复赛的主线。

AgentGuard Lab 的核心不在文案生成，而在“评测、失败定位、修复、复测”的工程闭环。这更贴合你的 AI Agent / AI Engineering 方向，也更容易展示技术含量。

## 核心用户与痛点

目标用户：

- 正在做课程助手、代码评审助手、知识库问答助手、营销助手等 AI Agent 的学生开发者。
- 需要展示 AI 应用可靠性的竞赛团队。
- 想让 Agent 从“能聊”变成“可测试、可维护、可交付”的项目作者。

真实痛点：

- Agent Demo 很容易只展示顺利样例，缺少失败样例和回归测试。
- 系统提示词、知识库、工具调用之间经常互相冲突，但开发者难以定位问题。
- 普通 AI 生成建议没有证据链，改完之后也不知道是否真的变好。
- 比赛或项目答辩时，评委很难判断 Agent 是否可靠、是否只是包装出来的聊天壳。

## 初赛 Demo 最小闭环

初赛不要做完整平台，只做 3 个内置样例和 1 条完整评测闭环：

1. 选择 Agent 样例：课程问答 Agent、代码评审 Agent、个人资料/作品集 Agent。
2. 输入 Agent 配置：系统提示词、知识库片段、工具说明或历史对话。
3. 生成测试集：事实性、边界拒答、工具调用、上下文污染、幻觉、指令冲突。
4. 运行评测：展示每条 case 的输入、输出、预期行为、实际结果和是否通过。
5. 失败归因：分类为知识缺口、提示词冲突、工具 schema 不清、边界策略缺失、上下文污染等。
6. 修复建议：生成 system prompt 修改建议、知识库补充建议、工具参数约束或拒答规则。
7. 复测对比：展示 before/after 通过率、失败项变化和导出报告。

## 评审维度匹配

创新性：

- 不是普通聊天助手，而是 Agent 质量保障工具。
- 把评测生成、失败归因、修复建议、复测对比做成闭环。

实用性：

- 直接解决 AI 应用开发中的可靠性问题。
- 适合学生竞赛、课程项目、开源 Agent、内部知识库助手等场景。

完成度：

- 初赛只需要完成可运行的评测工作台。
- 通过内置样例即可展示完整用户路径，不依赖复杂外部系统。

设计体验：

- 可以做成左侧 Agent 样例、中间评测运行、右侧失败报告、底部复测对比的仪表盘。
- 评委能一眼看懂“哪里失败、为什么失败、怎么修复、修复后是否变好”。

## 建议报名摘要

AgentGuard Lab 面向正在开发 AI Agent 的学生和团队，提供一个可交互的可靠性评测与修复工作台。它能根据 Agent 的提示词、知识库、工具说明或对话记录自动生成测试用例，运行事实性、边界、工具调用和上下文污染等评测，定位失败原因，并给出可复测的修复建议。它的价值在于让 Agent 项目不再停留于“能聊、能演示”，而是进入“可测试、可解释、可维护、可交付”的工程状态。
